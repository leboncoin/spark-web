# Table

A table displays data in rows and columns and enables users to navigate its contents via keyboard. It is built on [React Aria Table](https://react-aria.adobe.com/Table) and supports sorting, empty state, and dynamic collections.

## Install

```sh
npm install @spark-ui/components
```

## Import

```tsx
import {
  Table,
  useTableSort,
  useTablePagination,
  type SortDescriptor,
} from '@spark-ui/components/table'
```

## Anatomy

`Table` is the root container (a flex column with gap). It can wrap `Table.BulkBar`, `Table.Grid` (the data grid), and `Pagination`. `Table.Grid` is the scrollable table and requires `aria-label` or `aria-labelledby`.

```tsx
<Table>
  {/* Optional: Table.BulkBar when using selection + pagination */}
  <Table.Grid aria-label="Description of the table">
    <Table.Header>
      <Table.Column isRowHeader>Column 1</Table.Column>
      <Table.Column>Column 2</Table.Column>
    </Table.Header>
    <Table.Body>
      <Table.Row id="row-1">
        <Table.Cell>Cell 1</Table.Cell>
        <Table.Cell>Cell 2</Table.Cell>
      </Table.Row>
    </Table.Body>
    {/* When using items + renderEmptyState, provide the empty content; it will be wrapped for consistent padding */}
    {/* <Table.Body items={[]} renderEmptyState={() => 'No results.'}> */}
  </Table.Grid>
  {/* Optional: Pagination */}
</Table>
```

## API Reference

### Table

Root wrapper. Renders a flex column container (`gap-md flex flex-col`) for `Table.BulkBar`, `Table.Grid`, and optional siblings like `Pagination`. Pass selection props (`selectionMode`, `selectedKeys`, `onSelectionChange`), sort (`sortDescriptor`, `onSortChange`), layout (`allowsResizing`, `maxHeight`, resize callbacks), and BulkBar options (`totalCount`, `hasMultiplePages`, `onSelectAll`) on the root so they are provided via context to `Table.Grid` and `Table.BulkBar`. Accepts `className` and `children`.

<ArgTypes of={Table} />

### Table.Grid

The data grid (scroll container + semantic table). Requires `aria-label` or `aria-labelledby` for accessibility. Consumes selection, sort, and layout from Table context. Accepts optional `className` for the scroll wrapper.

<ArgTypes of={Table.Grid} />

### Table.Header

Defines the table header row and columns.

<ArgTypes of={Table.Header} />

### Table.Column

Defines a column header. Use `isRowHeader` for the first column when it labels each row. Use `allowsSorting` and pass `sortDescriptor` / `onSortChange` on `Table` (root) for sortable columns.

<ArgTypes of={Table.Column} />

### Table.Body

Contains the table rows. Prefer rendering rows with a simple `.map()` over your data array so that parent state updates flow correctly. For empty state, either render a single row with your own JSX when the list is empty, or use `items` with `renderEmptyState` (the content you return is automatically wrapped for consistent padding). For large tables where row content only depends on specific values, see **Row cache (items + dependencies)** below.

<ArgTypes of={Table.Body} />

### Table.Row

A table row. Requires a unique `id` for selection and keyboard navigation.

<ArgTypes of={Table.Row} />

### Table.Cell

A table cell.

<ArgTypes of={Table.Cell} />

### Table.BulkBar

Toolbar shown when using selection (e.g. with pagination). Renders a bar with selected count, clear/select-all actions, and custom actions. Must wrap `Table.BulkBarSelectedCount`, and optionally `Table.BulkBarClearButton`, `Table.BulkBarSelectAllButton`, and other controls. Selection state and BulkBar options are read from Table context (pass `totalCount`, `hasMultiplePages`, `onSelectAll` on `Table` when using pagination).

<ArgTypes of={Table.BulkBar} />

### Table.BulkBarSelectedCount

Displays the number of selected items. Use as a child of `Table.BulkBar`; provide the wording via `children` (e.g. "3 elements selected").

<ArgTypes of={Table.BulkBarSelectedCount} />

### Table.BulkBarClearButton

Button to clear the current selection. Renders only when `hasMultiplePages` is true. Use as a child of `Table.BulkBar`; provide the label via `children` (e.g. "Clear all").

<ArgTypes of={Table.BulkBarClearButton} />

### Table.BulkBarSelectAllButton

Button to select all items across pages. Renders only when `hasMultiplePages` is true and not all items are selected. Use as a child of `Table.BulkBar`; provide the label via `children` (e.g. "Select all 42 items").

<ArgTypes of={Table.BulkBarSelectAllButton} />

## Examples

### Sorting

Set `allowsSorting` on columns and pass `sortDescriptor` / `onSortChange` on `Table` (root) to enable sorting. Use the `useTableSort` hook to manage sort state and get a sorted list: it handles the descriptor state and a default comparator (numeric for numbers, `localeCompare` for strings).

<Canvas of={stories.Sortable} />

For columns that are not plain strings or numbers (e.g. date strings like `11/05/2024`), the default comparator can order incorrectly because it uses string comparison. Pass a custom `compare` to `useTableSort` to sort chronologically or by any other logic. In the example below, dates in `DD/MM/YYYY` format are parsed and compared as timestamps so that 11/05/2024 appears before 08/01/2025 when sorted ascending.

### Links

Pass `href` (and optionally `target`, `rel`, or `routerOptions`) to `Table.Row` to turn each row into a link while keeping checkbox selection. Use `target="_blank"` with `rel="noreferrer noopener"` to open links in a new tab safely.

Additionally, you can pass `onAction` to `Table.Row` to handle the action when the row is clicked.

<Callout kind="warning" marginY="large">
  <p>
    <strong>Accessibility:</strong> When row selection and row links are used together, React Aria
    disables link behavior as soon as at least one row is selected. This avoids conflicting
    interactions (click to select vs. click to navigate) and keeps a single, predictable primary
    action—selection takes precedence so users can use checkboxes without accidentally following a
    link.
  </p>
</Callout>

### Disable column resizing

Column resizing is enabled by default. Set `allowsResizing={false}` on `Table` (root) to disable it entirely, or `allowsResizing={false}` on individual `Table.Column` to hide the resizer for that column. Pass `onResizeStart`, `onResize`, or `onResizeEnd` on `Table` to react to resize events.

<Canvas of={stories.WithResizingDisabled} />

### Custom interactive elements

You can pass **children** to `Table.Column` to add actions next to the column label (e.g. a Popover trigger for info). Use **children** in `Table.Cell` to render interactive components such as Switch, buttons, or links.

<Canvas of={stories.CustomInteractiveElements} />

### Row cache (items + dependencies)

For large tables or when the parent re-renders often for unrelated state (e.g. a "last clicked" label, a search input), using `items` plus `dependencies` lets React Aria cache each row: the render function runs only when `dependencies` change. List in `dependencies` any value your row render function closes over (e.g. the data array, or external state like toggles). When those do not change, row content is not re-rendered. Use this when you need to limit re-renders for performance.

<Canvas of={stories.WithItemsAndDependencies} />

### Pagination

Combine the Table with Spark's [Pagination](/docs/components-pagination--docs) to display large datasets page by page. Hold the full dataset in state, and use the `useTablePagination` hook to derive the current page, paginated items, and a selection model that persists across pages. Pass `selectedKeys`, `onSelectionChange`, `totalCount`, `hasMultiplePages`, and `onSelectAll` on `Table` (root); wire `page` / `onPageChange` to the Pagination. The bulk bar reads selection state from context. The example below shows 134 items with 10 items per page (`pageSize={10}`).

When using `Table.BulkBar` with pagination, pass `hasMultiplePages={totalItems > pageSize}` and `onSelectAll={() => onSelectionChange(allKeys)}` on `Table`. The bulk bar is always visible; only the clear and select-all buttons are conditional. Provide all wording via `children` on `Table.BulkBarSelectedCount`, `Table.BulkBarClearButton`, and `Table.BulkBarSelectAllButton`—the components do not render any hardcoded copy.

## Accessibility

- The table uses semantic `<table>`, `<thead>`, `<tbody>`, `<tr>`, `<th>`, and `<td>` elements. For interactive navigation, React Aria uses `role="grid"` on the table element.
- Always provide an `aria-label` (or `aria-labelledby`) on `Table.Grid`.
- For i18n, Table supports overriding the few built-in accessible labels (see below).
- Use `isRowHeader` on the first column when it describes each row.
- Keyboard: Arrow keys navigate between cells; Enter activates row action when using `onRowAction`.
- See [React Aria Table](https://react-aria.adobe.com/Table) for full keyboard and screen reader behavior.

### Accessible labels (i18n)

Table’s public API exposes label overrides so consumers can provide localized copy:

- **`Table.Grid`**: pass `aria-label` or `aria-labelledby` (required).
- **`Table` (root)**:
  - `resizeColumnAriaLabel`: `string | (column) => string` — label used by the column resize control.
- **`Table.BulkBar`**: pass an `aria-label` prop to label the bulk actions toolbar.

```tsx
<Table
  resizeColumnAriaLabel="Resize column"
>
  <Table.BulkBar aria-label="Table bulk actions">
    {/* ... */}
  </Table.BulkBar>
  <Table.Grid aria-label="Files">
    {/* ... */}
  </Table.Grid>
</Table>
```

### Keyboard navigation

The Table follows React Aria’s default [Table](https://react-aria.adobe.com/Table) keyboard behavior (WAI-ARIA Grid pattern).

- **Arrow keys**: move focus within the grid (cells / headers) per React Aria.
- **Selection**: when enabled, <Kbd>Space</Kbd> toggles selection following React Aria rules.
- **Sorting**: sortable headers can be triggered from the header as provided by React Aria.
- **Interactive content in cells**: Spark adds an explicit distinction between **grid navigation** and **interaction** inside body cells so keyboard users can both navigate the table and operate controls inside cells.

#### Grid mode vs interaction mode (body cells)

By default, the table is in **grid mode**:

- Focus stays on the active cell (`td`), and Arrow keys move between cells (React Aria roving focus).
- Focusable descendants inside body cells are temporarily removed from the tab order so grid navigation does not “fall into” buttons/links/inputs.

When a focused body cell contains interactive controls, you can enter **interaction mode**:

- Press <Kbd>Enter</Kbd> or <Kbd>F2</Kbd> to move focus to the first focusable element inside the current cell.
- While in interaction mode, <Kbd>Tab</Kbd> moves between focusable elements inside that cell (native tab order).
- Press <Kbd>Escape</Kbd> (or <Kbd>F2</Kbd>) to exit interaction mode and return focus to the cell.

Notes:

- The **selection column** (row checkbox) does not enter interaction mode; it keeps grid navigation consistent.
- For comboboxes/dropdowns, <Kbd>Escape</Kbd> first closes the open popup (APG pattern). Exiting interaction mode happens only once the popup is closed.
