# SegmentedControl

A linear set of mutually exclusive options. Selecting one deselects all others. An optional sliding indicator visually tracks the active selection.

It is primarily designed as a **navigation control** (e.g. switching calendar views), but can also be used as a form input when the number of options is small and the compact, horizontal layout is appropriate.

<Canvas of={stories.Default} />

## Install

```
npm install @spark-ui/components
```

## Anatomy

```tsx
import { SegmentedControl } from '@spark-ui/components/segmented-control'

export default () => (
  <SegmentedControl defaultValue="a">
    <SegmentedControl.Indicator />
    <SegmentedControl.Item value="a">Option A</SegmentedControl.Item>
    <SegmentedControl.Item value="b">Option B</SegmentedControl.Item>
    <SegmentedControl.Item value="c">Option C</SegmentedControl.Item>
  </SegmentedControl>
)
```

## API Reference

### SegmentedControl

The root container. Renders as `role="radiogroup"`.

<ArgTypes of={SegmentedControl} />

### \_.Item

A single selectable option. Renders as `role="radio"`.

<ArgTypes of={SegmentedControl.Item} />

### \_.Indicator

An animated rectangle that slides to the active item. Rendered absolutely inside the root container.

## Form field

SegmentedControl integrates with `FormField` to receive an accessible label, helper text, and validation messages. This is the recommended approach when using it as a form input.

<Callout kind="warning">
  Use SegmentedControl as a form input only when all of the following apply:
  <ul className="pl-3xl">
    <li className="list-disc">
      There are **2 to 4 short options** — for more options, prefer `RadioGroup`.
    </li>
    <li className="list-disc">
      The horizontal, compact layout provides a clear benefit in context.
    </li>
    <li className="list-disc">Labels are concise (a single word or short phrase).</li>

## Accessibility

- The root element renders as `role="radiogroup"`.
- Each item renders as `role="radio"` with `aria-checked` managed by Base UI.
- Keyboard navigation: **Tab** moves focus into the group, **Arrow keys** move between items (roving tabindex — only one Tab stop for the entire group).
- Icon-only items must have an accessible label via `aria-label`.
- The `SegmentedControl.Indicator` is `aria-hidden` and purely decorative.
- When used inside `FormField`, the group is automatically labelled via `aria-labelledby` and described via `aria-describedby`.
- When not inside `FormField` (navigation use case), provide an accessible name via `aria-label` or `aria-labelledby` directly on `SegmentedControl`.
