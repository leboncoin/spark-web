# Card

A card is a generic container with multiple design and intent options. It is non-interactive by default but can be turned into either a link or a button.

<Canvas of={CardStories.Default} />

## Install

```sh
npm install @spark-ui/components
```

## Anatomy

```js
import { Card } from '@spark-ui/components/card'

export default () => (
  <Card>
    <Card.Type>Featured</Card.Type>
    <Card.Content>{/* Your content here */}</Card.Content>
  </Card>
)
```

The Card component consists of three main parts:

- **Card**: The root container that provides the card's styling and context (with `role="article"`)
- **Card.Type**: An optional gradient banner with text to indicate the type or category of the card
- **Card.Content**: The content container with proper padding and spacing

## Examples

### Type

The `Card.Type` component displays a text label on a gradient background at the top of the card. Use it to indicate the card's type, category, or status.

**Common use cases:**

- Product categories: "Best seller", "New product", "Limited edition"
- Status indicators: "Featured", "Important", "Archived"
- Loading states: "Loading..." (with `animation="pulse"`)

## Advanced Examples

### Link

Use `asChild` so that the card takes the html tag and props of its direct child element.
In this example, if you need your card to be a link.

This polymorphic behaviour can be used with any type of HTML element, but the following might be more common: `div, article, section, aside, button, a, form, li, nav`

<Callout kind="warning">
  If you do decide to change the underlying element type, it is your responsibility to ensure it
  remains accessible and functional. In the case of Tooltip.Trigger for example, it must be a
  focusable element that can respond to pointer and keyboard events. If you were to switch it to a
  div, it would no longer be accessible.
</Callout>

### LinkBox

In case you wish to have interactive elements inside the card WHILE the card itself is a link, you can use the `LinkBox` component.

It will keep the HTML valid and accessible (as it is forbidden to have nested links, but `LinkBox` is a wrapper that allows it).

## Accessibility

### Semantic HTML

Card uses `role="article"` to identify itself as a standalone article for screen readers. Card.Type uses a `<header>` element with non-heading text (not `<h1>`-`<h6>`) to avoid conflicts with the card's main title.

### Heading Hierarchy

When using Card.Type with content that includes headings, follow this pattern:

```jsx
<main>
  <h1>Page Title</h1>

  <Card role="article" aria-labelledby="card-title-1">
    <Card.Type>Best seller</Card.Type> {/* Not a heading - just a label */}
    <Card.Content>
      <h2 id="card-title-1">Product Name</h2> {/* Main card title */}
      <p>Description...</p>
      <h3>Features</h3> {/* Subsection */}
    </Card.Content>
  </Card>
</main>
```

**Key points:**

- Card.Type displays a type/category label, not a hierarchical title
- The main title should be inside Card.Content as an `<h2>` or `<h3>` (depending on page structure)
- Use `aria-labelledby` to point to the main title for proper card identification
